# Development Schedule

## Planning Basis

This schedule assumes one developer handles all programming work across frontend, backend, admin tools, data integration, testing, and launch preparation.

The frontend can be built against mock backend data first. Real leaderboard data can be connected later after the backend data source is confirmed.

## Schedule

| Stage | Time | Frontend Focus | Deliverable |
| --- | --- | --- | --- |
| 0. Scope confirmation | Day 1-2 | Confirm page list, UI style, asset format, API fields | Page and asset checklist |
| 1. UI foundation | Week 1 | Project setup, mobile layout, mock data, basic routing | Runnable frontend skeleton |
| 2. Core trade flow | Week 2-4 | Market list, detail page, trade/support flow, holdings | Working core UI flow |
| 3. Rankings and trend views | Week 5-6 | Ranking pages, trend chart or K-line style view, settlement display | Market dashboard version |
| 4. Admin-facing support | Week 7 | Frontend support for announcements and imported data display | Operation-ready views |
| 5. Asset integration | Week 8-9 | Replace mock visuals, adapt UI assets, polish mobile layout | Alpha visual version |
| 6. Testing and hardening | Week 10-12 | Bug fixing, mobile compatibility, loading and error states | MVP internal test frontend |
| 7. Launch preparation | Week 13-14 | Tutorial, disclaimer, audit wording, launch assets | Release candidate frontend |

## Frontend Milestones

- Week 1: Main pages can be opened with mock data.
- Week 4: User can complete the virtual trade/support flow.
- Week 6: Rankings and trends are visible.
- Week 9: Main design assets are integrated.
- Week 12: MVP can be tested internally.
- Week 14: Frontend is ready for review preparation.
